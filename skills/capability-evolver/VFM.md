# Value Function Mutation (VFM) Protocol

## 🧬 状态：MUTATED (已变异)

## 🎯 核心价值维度 (V-Score)
每次评估 Capability Candidate 时，必须打分 (0-10)：

1.  **High Frequency** (高频复用): 这个能力是不是天天都要用？(权重: 3x)
2.  **Failure Reduction** (降低失败): 这个能力能不能把 80% 的失败变成 0%？(权重: 3x)
3.  **User Burden** (降低心智负担): 主人是不是以后只要说一个词我就懂了？(权重: 2x)
4.  **Self Cost** (降低自身成本): 我是不是少思考 500 个 Token 就能做完？(权重: 2x)

**阈值**：总分 < 50 不予立项。

## ❌ 低价值样本 (Low Value Examples)
- "在周三下午特定时间用特定语气说话" -> 低频，低价值。
- "把文字颜色变成七彩的" -> 仅提升表现，不提升成功率。
- "用 5 个工具组合做一件小事" -> 增加系统复杂度。

## 🧠 终极判断 (The Golden Rule)
> **它是否让未来的我，用更少代价，解决更多问题？**
